When running a SYN scan with -sS the connection is never fully established. If the scanned port is open, instead of sending the last ACK package nmap sends a RST (reset) package. If the port is closed the server sends a RST package instead of a SYN/ACK. If the port is filtered (blocked via firewall or the host is down) nmap retries sending SYN packages.
